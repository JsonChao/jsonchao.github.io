android常用布局及控件的使用

Hello，这是android基础的第一堂课

1、layout_gravity:设置的是本视图相对于上一级视图的显示位置

对于水平方向的线性布局,因为每个视图是按照出现的先后顺序单独占据一列，所以不可以使用layout_gravity
改变视图在水平方向的位置，比如，left，right,center_horizontal
对于垂直方向的线性布局,因为每个视图是按照出现的先后顺序单独占据一行，所以不可以使用layout_gravity
改变视图在垂直方向的位置，比如，top，bottom,center_vertical

2、TextView

TextView的常用属性

RGB:每个颜色值是一个一位的16进制的数,最小值是0,最大值是f
RRGGBB:每个颜色值是一个两位的16进制的数,最小值是00,最大值是ff
ARGB:A表示透明度A是一个1位的16进制数,A的值是0表示完全透明,A的值是f表示完全不透明
AARRGGBB:AA表示透明度，AA是一个2位的16进制数,AA的值是00表示完全透明，A的值是ff表示完全不透明
单行显示文本android:singleLine="true"
android:ellipsize:在起始，中间，结束位置显示省略号
实现跑马灯效果
首先需要单行显示
android:singleLine="true"
android:ellipsize="marquee"
TextView 默认情况下是没有获得焦点的，
也就是这两个属性android:focusable,android:focusableInTouchMode的默认值是false
android:focusable:针对按键的
android:focusableInTouchMode:针对屏幕触摸的
ndroid:focusable = "true"是android:focusableInTouchMode="true"有效的前提条件

3、EditText

android:inputType:帮助输入法确定显示什么样的键盘
android:adjustViewBounds:当ImageView的宽高发生变化时，图片按比例缩放,该属性必须结合着android:maxWidth,android:maxHeight这两个属性使用

4、imageview 

scaleType:设置图片的填充方式
center:按照原图大小显示，但是当图片的宽高大于ImageView的宽高时，截取图片中间部分显示
centerCrop:按比例缩放原图，直到等于ImageView的宽高时显示
centerInside:图片的宽高等于ImageView的宽高时，按照原图大小居中显示，否则将图片缩放至ImageView的大小并居中显示
fitXY:拉伸图片（不按比例），填充ImageView
fitStart:按比例拉伸图片，拉伸后图片的高度为ImageView的高度，并显示在ImageView的左边
fitCenter:按比例拉伸图片，拉伸后图片的高度为ImageView的高度，并显示在ImageView的中间
fitEnd:按比例拉伸图片，拉伸后图片的高度为ImageView的高度，并显示在ImageView的右边

7、ActionBar

在清单文件中加入ActionBar分隔栏:

<activity
    android:name="com.qianfeng.gp10_day30_actionbar1.MainActivity"
    android:label="@string/app_name" 
    android:uiOptions="splitActionBarWhenNarrow"
    >

启用应用导航—显示回退到上一个Activity的箭头让导航显示

getActionBar().setDisplayShowHomeEnabled(true);
//让该动作项可以被点击---该动作项的id是android.R.id.home
getActionBar().setDisplayHomeAsUpEnabled(true);
//ActionBar的自动隐藏和显示
@Override
public boolean onTouchEvent(MotionEvent event) {
    // TODO 处理屏幕触摸事件:按下 移动 抬起
    if(event.getAction()==MotionEvent.ACTION_UP)
    {
        if(getActionBar().isShowing())//判断ActionBar是否显示
        {
            getActionBar().hide();//隐藏
        }
        else
            getActionBar().show();
    }
    return true;//表示处理当前的触摸事件
}
