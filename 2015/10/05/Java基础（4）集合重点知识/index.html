Java基础（4）集合重点知识

Posté le 2015-10-05 | In 安卓基础 | Visitors:

Hello，这是java的第四堂课

1、集合变数组：

Object[] toArray()

2、iterator()方法返回的是实现了Iterator接口的内部类对象，而内部类中提供了hasNext()和next()方法，从而实现集合的遍历，在遍历集合的过程中使用集合对集合中的对象进行添加，删除，修改会发生ConcurrentModificationException异常

Enumeration enume = v.elements();
    while(enume.hasMoreElements())
    {
        System.out.println(enume.nextElement());
    }

3、LinkedList中的特有方法:

addFirst()
addLast()

getFirst()//获取的对象不存在，则会异常.NoSuchElementException
getLast()  //获取的对象不存在，则会异常.NoSuchElementException

removeFirst()//删除的对象不存在，会异常NoSuchElementException
removeLast()

从jdk1.6开始出现的功能
offerFirst()
offerLast()

peekFirst()//获取的对象不存在，返回null，不会异常
peekLast()

pollFirst()///删除的对象不存在,不会异常，返回null
pollLast()

4、哈希值相同不一定是同一个对象,同一个对象的哈希值肯定相同

5、TreeSet:底层使用的数据结构是二叉树，是线程不安全的

6、对集合中的对象进行排序的第一种方式:集合中的对象所属的类实现了Comparable接口中
的int compareTo(Object obj)方法,保证对象唯一的方式就是：compareTo方法的返回值为0

第二种方式：自定义一种排序方式(即自定义一个排序类)，该方式(类)要实现 Comparator接口中的
int compare(Object obj1,Object obj2)方法,把这种排序方式作为参数传递给
集合的构造方法

注意：当排序的主要条件相等时，还需要指定排序的次要条件

7、TreeSet因为 String类实现了Comparable接口中的int compareTo()方法，所以集合是依据该方法的返回值来判断谁大谁小的

TreeSet<Student> ts1 = new TreeSet<Student>();
        ts1.add(new Student("lisi",23));//TreeSet在添加的时候就进行了比较,排序
        ts1.add(new Student("zhangsan",22));
        ts1.add(new Student("wangwu",27));

8、TreeSet(Collection<? extends Person> c) 

TreeSet<Person>  ts = new TreeSet<Person>(ts1);

9、HashMap:底层使用的数据结构是哈希表，线程不安全的，可以存储null键，null值

TreeMap:底层使用的数据结构是二叉树，线程不安全的

10、Map的获取功能

Set<K> keySet() 
Set<Map.Entry<K,V>> entrySet() 
返回的是上一次添加的值

11、sop(map.put(“name”,”lisi”));//第一次添加返回值为null

12、Map中没有Iterator迭代器,有以下两种方式转为Set并使用Set中的Iterator

Set<Map.Entry<String,String>> en = map.entrySet();
Iterator<Map.Entry<String,String> ite = en.iterator();
while(ite.hasNext())
{
    Map.Entry<String,String> entry = ite.next();
    String key = entry.getKey();
    String value = entry.getValue();
    System.out.println(key+"="+value);
}
Set<String> set = map.keySet();
Iterator<String> ite = set.iterator();
while(ite.hasNext())
{
    String key = ite.next();
    String value = map.get(key);
    System.out.println(key+"="+value);
}

13、根据键来排序的,排序的方式和 TreeSet一样

TreeMap<Student,String>  tm = new TreeMap<Student,String>(new ComByName());

自定义的排序方式优先

14、把数组中的数据转成字符串

String ss = Arrays.toString(arr);

15、把数组转成集合

String[] array ={"haha","hehe","xixi"};
List list = Arrays.asList(array)

16、数组转成的集合不允许进行添加，删除操作，因为数组的长度是固定的

list.add("huhu");//UnsupportedOperationException
list.remove("haha");//UnsupportedOperationException
int[] b = {1,2,3};
List list2 = Arrays.asList(b);
System.out.println(list2.size());//1因为集合添加的是引用类型的对象，因为数组是引用类型的，所以把整个数组作为一个对象添加到集合中

17、集合转数组 

<T> T[] toArray(T[] a)

18、数组的长度大于需要的长度，会直接使用该数组，数组的长度小于需要的长度，会创建一个新的数组，所以最好传递一个长度正好的数组

String[] arr = list.toArray(new String[list.size()]);

19、为什么集合转数组?

因为数组的长度是固定的，当不允许用户对数据随意的添加删除时，可以集合转数组

20、根据集合中的对象所属的类自身具备的排序方式进行排序

Collections.sort(list);

21、根据自定义的排序方式对集合中的对象进行排序

Collections.sort(list,new ComByLength());

22、按照集合中对象自身具备的排序方式的相反的方式进行排序

Collections.sort(list,Collections.reverseOrder());

23、倒序集合中的对象

Collections.reverse(list);
