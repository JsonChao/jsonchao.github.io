Java基础（5）多线程重点知识

Veröffentlicht am 2015-10-12 | in 安卓基础 | Visitors:

Hello，这是java的第五堂课

1、可变参数:必须写在参数列表的最后

public static int add(int b,int... a)//把传递过来的数据封装到数组中
{
    int sum =0;
    for(int i=0;i<a.length;i++)
    {
       sum = sum+a[i];
    }
    return sum;
}

2、每个 Java 应用程序都有一个 Runtime 类实例,通过该类的实例可以得到运行时相关的信息

Runtime runtime = Runtime.getRuntime();
runtime.exec("notepad C:\\Users\\qq\\Desktop\\gp04.txt");
long size = runtime.maxMemory();
System.out.println(size/1024/1024);

3、多线程一定能提高效率吗？

不一定能提高效率，但是能合理的使用cpu资源，最大的好处就是能够实现在一个应用中相同功能或不同功能的同时执行

4、每个对象都可以被回收—被回收的功能定义在Object类中

5、创建线程的第一种方式：

1：创建一个继承了Thread类的子类
2: 重写Thread类中的run方法
         创建线程是为了执行任务，任务代码需要有存储位置
         run方法就是这个存储位置
3: 创建子类的对象，实际就是创建线程
4：启动线程

6、线程任务和线程是绑定在一起的,使用该类创建的对象只能执行固定的任务

7、子线程的名字默认是：Thread-编号(编号从0开始)

主线程默认的名字:main

8、为了让四个线程共享50张票，所以把票修饰为static的

9、创建线程的第二种方式：

1：定义一个类实现Runnable接口
2: 该类重写Runnable接口中的 run方法
3：创建实现了Runnable接口的子类的对象
4: 创建 Thread类的对象，其实就是在创建线程
5: 把实现了Runnable接口的子类的对象作为参数传递给Thread类的构造方法

创建线程第二种方式的特点：

1:线程任务和线程对象进行了分离，线程执行什么任务不再重要，只要是实现了Runnable接口的子类对象，就可以作为参数传递给Thread的构造方法
2:实现 Runnable接口的同时，还可以继承父类

10、程序出现了线程安全问题：
打印了0,-1,-2等错票

出现问题的原因：

1：操作了共享数据
2: 操作共享数据的语句有多条，一个线程得到cpu，执行了操作共享数据的多条语句的一部分时，还没有全部执行完cpu就被抢走了

解决问题的方式：一个线程得到cpu，执行操作共享数据的多条语句时，其它线程不能参与执行，只有当该线程把所有操作共享数据的语句全部执行，其它线程才能参与执行

java提供的解决方式：

同步代码
synchronized(对象)
{
    需要被同步的代码(操作共享数据的语句)
}

12、静态同步函数使用的锁是什么？静态进内存的时候是没有对象的，但是肯定有其所属类的字节码文件，属于Class类型的对象。所以静态同步函数使用的锁是其所属类的字节码文件

13、单例设计模式中懒汉式并发访问的安全问题

class Single
{
    private static final Single s = new Single();

    private Single(){}

    public static Single getInstance()
    {
        return s;
    }
}
class Single2
{
    private static Single2 s;
    private Single2(){}
    public static Single2 getInstance()
    {
        if(s==null)//至少从第三个线程开始就不用判断锁了
        {
            synchronized(Single2.class)//1  2
            {
                if(s==null)
                    //---->
                    s = new Single2();
            }
        }
        return s;
    }
}

14、线程间的通信:多个线程执行的任务不同，但是操作的数据相同

15、为什么wait(),notify(),notifyAll()定义在Object类中？

因为这些方法必须用在同步中，同步中才有锁，而锁是任意的对象，任意的对象都可以调用的方法只能定义在Object 中

try{res.wait();}catch(InterruptedException e){}//等待的线程会放弃锁

16、jdk1.5多线程的实现方式

jdk1.5之前对锁的操作是隐式的

synchronized(对象)//获取锁
{

}//释放锁

jdk1.5对锁的操作是显式的

17、有一个单独的接口来描述锁,这个接口就是Lock,所以把锁面向对象了，使用Lock的子类ReentrantLock替代同步代码块

1:创建ReentranLock类的对象，就相当于创建了一把锁
2:把需要同步的代码放在 lock()和unlock()之间

使用Lock的子类ReentrantLock替代同步代码块之后，出现了IllegalMonitorStateException异常，原因是wait(),notifyAll()这些方法必须用在同步中，而同步没有了造成的有一个Condition接口，是对await(),signal(),signalAll()这些方法的描述
所以这些唤醒等待方法也被面向对象了，但是这些方法的使用必须指明锁，所以通过Lock的newCondition()方法来得到一个和锁绑定的Codition对象,从而间接的指明锁

17、线程的停止：stop()方法已经过时，所以只能是线程的任务代码执行完，线程才能停止，但是通常任务代码中都有
循环，所以只要让循环停下来就能让线程停止

18、在任务（同步）代码中定义一个changeFlag的方法，能够把标记改为相反的值,然后在主线程中设定要循环的次数

t1.interrupt();//结束线程的等待状态，让线程可以重新去争抢cpu, 从而就能执行任务代码了

19、守护线程：看成后台线程，依赖于前台线程，当前台线程全部结束时，即使守护线程的任务代码没执行完，也会停止

t2.setDaemon(true);//把t2线程设置为守护线程,,在启动之前设置为守护线程

20、join: 让线程加入运行

t1.join();
